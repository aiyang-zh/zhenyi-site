# 你写过什么不错的开源项目？

我长期维护的是 **zhenyi**（含 `zhenyi-base`）这一组开源项目。  
下面是 zhenyi 的核心技术能力清单，方便你快速判断是否适合自己的项目。

## zhenyi 的亮点能力（条目版）

1. **实时架构主线完整**  
   Actor 消息驱动 + Reactor 事件驱动网络 + 统一网关 `zgate`，把连接接入、会话管理、消息派发和业务执行串成一条链路。

2. **多协议长连接接入**  
   支持 TCP / WebSocket / KCP 等接入能力，适合游戏服、IM、推送、在线协作这类持续在线场景。

3. **服务级 Actor 并发模型**  
   用消息驱动组织状态更新，减少共享状态竞争和手写锁复杂度，适合玩家/房间/会话这类高并发状态场景。

4. **跨平台 Reactor 网络模型**  
   Linux 与 macOS 都可走事件驱动模式，配合少量 goroutine 支撑大量连接，强调连接规模与运行稳定性。

5. **高性能消息通路**  
   关键路径强调低分配、对象复用、批处理发送等策略，目标是降低 GC 扰动并提升吞吐利用率。

6. **高并发队列与基础组件**  
   提供无锁队列（如 `zqueue`）、对象池、序列化、限流、错误体系等基础能力，减少业务重复造轮子。

7. **可观测与告警能力**  
   Prometheus 指标、慢路径提示、链路追踪相关能力可接入，方便定位吞吐下降、延迟升高、队列积压等问题。

8. **分布式协作能力**  
   提供服务发现、跨进程消息总线、远程路由相关能力，支持从单机到多节点的架构演进。

9. **可运行示例与压测路径**  
   提供 IM 示例、压测客户端等工程化样例，方便先跑通再按业务改造。

10. **文档与 API 导航完善**  
    有成体系文档与示例索引，降低接入、扩展和二次开发成本。

11. **信创与国密方向支持**  
    提供国密 TLS（SM2/SM3/SM4）相关能力，便于有合规要求的项目评估和落地。

12. **面向生产的持续迭代**  
    不只关注 demo 可跑，更关注长时间稳定运行、可排障、可优化、可扩展。

## 适用场景

- MMORPG / 房间服 / 大厅服
- IM / 实时消息网关
- 在线协作 / 实时推送系统
- 需要长连接、高并发、低延迟的后端服务
